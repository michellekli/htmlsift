# Coding practices

1. First, check for relevant tests. Create them if they don't already exist.
2. Check implementation correctness by ensuring unit tests succeed with "uv run python -m unittest".
3. Narrow down issues by checking against the failing tests.
4. Follow coding best practices: KISS, DRY, and YAGNI.
  - KISS: Keep It Simple, Stupid
  - DRY: Don't Repeat Yourself
  - YAGNI: You Aren't Gonna Need It
5. Add documentation according to prompts/document.md
6. Don't remove any comments.
